# SmartVulnLifecycle
自动化智能合约漏洞全生命周期管理
## 项目介绍

VulnerabilityLifecycleManagement 是一款针对**智能合约漏洞全生命周期**的自动化管理系统，提供「漏洞检测 → 漏洞验证 → 漏洞修复 → 回归验证」的端到端闭环解决方案。系统聚焦区块链智能合约（支持 Ethereum 等多链）的安全管理需求，通过多智能体协作、自动化 PoC 生成、语义迁移修复等核心技术，解决传统漏洞管理流程分散、验证复杂、修复效率低的痛点，为开发者、安全审计人员提供统一的漏洞管理工作台。

核心目标：让智能合约漏洞的发现、验证、修复全流程自动化、可视化、可追溯，降低安全审计门槛，保障区块链资产安全。

## 核心功能

### 1. 基于多智能体审计的漏洞扫描（Scan Center）
- **灵活扫描配置**：支持按链（如 Ethereum）、协议、合约族（Proxy/ImplLibrary）筛选扫描范围，同步链上合约信息与代理关系。
- **多引擎协同**：整合静态分析（规则+AST）、符号执行、引导式模糊测试（Fuzz）、差分扫描（合约升级前后对比）等多引擎，智能体自动分解并行任务。
- **动态扫描策略**：根据合约风险画像（权限密集度、外部调用频率、资产可转移性）动态分配扫描深度与超时预算，平衡效率与准确性。
- **扫描结果输出**：生成带根因标签、置信度评分、影响范围的候选漏洞，支持一键转工单、去重合并已有工单。

### 2. 基于自动化 PoC 生成的漏洞验证（Verification Lab）
- **全维度漏洞信息展示**：清晰呈现漏洞类型、触发路径、依赖条件、风险等级（High/MED）、关联合约与函数。
- **自动化 PoC 生成**：基于 Foundry/Hardhat 框架生成可直接执行的漏洞利用脚本，支持复制、下载与二次开发。
- **交易语义还原**：将执行日志（tracelog）还原为「攻击叙事」，包含交易时间线、关键状态差分提示，直观理解漏洞利用过程。
- **可复现证据链**：支持 Fork 链上区块（如 `19,233,001`）固化测试环境，运行 PoC 后生成包含编译、部署、回放、断言结果的完整证据包。
- **PoC 迭代优化**：支持用户输入自定义约束，自动化调整 PoC 生成逻辑，提升验证准确性。

### 3. 基于语义迁移的漏洞修复与回归验证（Patch Studio）
- **多方案补丁候选**：针对漏洞根因生成多种修复方案（如 ReentrancyGuard+CEI、纯 CEI 重排、Pull-Payment 延迟提款），标注安全等级、Gas 消耗、兼容性评估。
- **修复约束保障**：严格遵循「不改变业务语义（接口/事件保持兼容）、阻断漏洞利用、不引入新风险」的修复准则。
- **PoC 回归验证**：复用验证阶段的 PoC 脚本，自动对比补丁前后的漏洞触发结果，验证漏洞是否被阻断、资产是否安全。
- **修复效果可视化**：展示补丁 Diff 对比、PoC 执行日志差异、关键断言结果（如资产流出=0），支持导出验证报告。

### 4. 漏洞工单全生命周期管理（Vulnerability Ops）
- **统一工单入口**：所有检测、验证、修复动作汇聚于同一工单，形成完整证据链与审计流程。
- **生命周期状态追踪**：支持 `Detected → Reproducible → Confirmed → Patching → Verified → Closed` 状态流转，清晰掌握漏洞处理进度。
- **优先级提醒**：自动标记高危未闭环、已可复现未修复的工单，优先处理高风险漏洞。
- **多维度搜索筛选**：支持按合约地址、漏洞类型、工单 ID、函数名快速检索目标工单。

## 主要特性
- **闭环管理**：从检测到修复的全流程无断点，无需切换多工具，提升工作效率。
- **高度自动化**：多智能体编排扫描任务、自动化 PoC 生成、智能补丁推荐，减少人工干预。
- **语义感知修复**：基于语义迁移技术，确保修复不破坏合约原有业务逻辑与兼容性。
- **多链支持**：兼容 Ethereum 等主流区块链，支持多链合约画像与扫描。
- **可视化操作**：三栏布局工作台、直观的状态标识、详细的日志输出，降低使用门槛。
- **可追溯性**：完整记录漏洞处理全流程的操作日志与证据包，满足审计合规需求。

## 界面展示

### 主页面（Dashboard）
核心入口，展示所有漏洞工单列表、当前高危未闭环队列、今日自动化产出（扫描次数、PoC 数量、补丁候选数），支持快速跳转至扫描、验证、修复模块。

### 扫描中心（Scan Center）
- 左侧：合约上传与代码展示区
- 右侧：智能体选择与扫描策略配置区
- 下侧：扫描结果列表（漏洞类型、定位、置信度、操作按钮）

### 验证实验室（Verification Lab）
- 左侧：漏洞信息详情（根因、依赖条件、建议动作）
- 右侧：PoC 代码生成区（支持复制/下载）
- 底部：PoC 运行日志与证据链展示区

### 修复工作室（Patch Studio）
- 左侧：待修复漏洞详情与根因约束
- 右侧：补丁候选列表（多方案对比）
- 底部：PoC 回归验证结果与日志对比区

## 安装部署

### 环境要求
- Node.js ≥ 16.x
- npm/yarn
- Foundry/Hardhat（用于 PoC 执行与验证）
- 区块链节点 RPC 接口（如 Infura、Alchemy）

### 部署步骤
1. 克隆仓库
   ```bash
   git clone https://github.com/your-username/VulnerabilityLifecycleManagement.git
   cd VulnerabilityLifecycleManagement
   ```

2. 安装依赖
   ```bash
   npm install
   # 或
   yarn install
   ```

3. 配置环境变量
   复制 `.env.example` 为 `.env`，填写区块链 RPC 地址、API 密钥等配置：
   ```env
   RPC_URL=https://mainnet.infura.io/v3/your-api-key
   FORK_BLOCK=19233001
   DEFAULT_CHAIN=ethereum
   ```

4. 启动前端原型
   ```bash
   npm run start
   # 访问 http://localhost:3000 即可使用
   ```

## 使用指南

### 快速流程
1. **上传合约**：在 Scan Center 上传智能合约代码（或通过合约地址同步链上合约）。
2. **配置扫描**：选择目标链、扫描引擎与智能体，设置超时预算与路径上限，启动扫描。
3. **验证漏洞**：扫描完成后，将候选漏洞转工单，进入 Verification Lab 运行 PoC 验证漏洞真实性。
4. **修复漏洞**：进入 Patch Studio 选择合适的补丁方案，查看修复效果与影响评估。
5. **回归验证**：通过 PoC 回归测试确认漏洞已阻断，导出验证报告，关闭工单。
